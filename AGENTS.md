# Project Instructions

- Do not start or leave a local dev/preview server for this project unless the user explicitly asks.
- For browser verification, assume the user serves the app and use `http://localhost:3000`.
- If `http://localhost:3000` is not reachable, report that the user-run dev server is unavailable instead of starting another server.
- Use Bun for project commands such as `bun test` and `bun run build`.
- Do not attempt git commits. The user handles GPG authentication manually.
- Staging files is allowed only when requested; if files are already staged, confirm before staging additional files.
- Keep JSX/TSX implementation files under 500 lines unless there is a clear, documented reason for an exception.
